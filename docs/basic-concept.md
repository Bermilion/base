# Основная концепция

## CSS переменные
В пакете заранее определены переменные для реализации микромодулей в `scss/config/_sizes.scss` лежит `map` в котором хранятся ключи и значения величин приемлемых в интерфейсе:

```scss
$sizes: (
	'0': 0,       // 0px
	'1px': 0.125, // 1px
	'2px': 0.25,  // 2px
	'4px': 0.5,   // 4px
	'base': 1,    // 8px
	'xs': 1.5,    // 12px
	'sm': 2,      // 16px
	'md': 2.5,    // 20px
	'lg': 3,      // 24px
	'xl': 4,      // 32px
	'2xl': 5,     // 40px
	'3xl': 6,     // 48px
	'4xl': 7,     // 56px
	'5xl': 8,     // 64px
	'6xl': 9,     // 72px
	'7xl': 10,    // 80px
	'8xl': 11,    // 88px
	'9xl': 12,    // 96px
	'10xl': 13,   // 104px
);
```
Так как базовая величина шрифта определена в `scss/core/_rem.scss` равной `50%` микромодуль равен 8px = 1rem. Это ключ `base` из карты выше.

## Микромодуль
Микромодуль — это самая малая и неделимая часть сетки интерфейса. Микромодуль в интерфейсе не изображается, это условность и абстракция.

Размеры остальных элементов в интерфейсе, которые задаются вручную, кратны размеру микромодуля.

> Микромодуль в «Чанкере» — квадрат размером 8×8.

То есть размеры остальных элементов в интерфейсе кратны 8.

> Корректный размер: 8, 16, 24, 32, 40 и т.д. — они делятся на 8 без остатка.

> **Внимание!**
> 
> Некорректный размер: 6, 10, 12, 17, 19.93 и т.д. — они делятся на 8 с остатком.

Однако, случаются ситуации, когда 8 — слишком много. Например, в кнопке могут располагаться иконка и текст. И при пробеле в 8 они кажутся оторванными друг от друга. Тогда допустимо использовать значение меньше 8:

> Допустимые размеры меньше микромодуля: 4, 2, 1, 0.

## Польза микромодуля
Использование микромодуля даёт несколько преимуществ.

Дизайн выглядит более упорядоченным и гармоничным, а сетка подчиняется шаблону. В том числе формируется вертикальный ритм. Всё это снижает когнитивную нагрузку на пользователя, так как интерфейс и контент становится более простым для восприятия.

Упрощаются процессы дизайна и вёрстки, так как решение о выборе размера становится шаблонным. Дизайнер и верстальщик справляются с работой быстрее и допускают меньше ошибок, что экономит время и бюджет.

## Допустимые несоответствия
Есть ситуации, в которых размер элемента определяется автоматически, а не вручную.

Например: кнопка с текстом. Даже если размер текста и высота строки в кнопке кратны микромодулю, ширина букв и всего текста зависит от используемого шрифта, поэтому нормально, если ширина кнопки (или подобного элемента) не кратна 8 пикселям.

Другой такой ситуацией может быть применение колонок. Допустим, ширина страницы на смартфоне — 360. Боковые поля контента — 16. То есть ширина контента: 360 - 16 х 2 = 328. И дизайнер решает построить контент в 4 колонки с гаттером в 8: (328 - 8 * 3) / 4 = 74. Такая ширина колонки не кратна 8: 74 / 8 = 9,25, то есть она не подчиняется микромодулю. Такая ситуация допустима, так как в таком случае у четырехколоночной сетки есть приоритет, и она на ломает вертикальный ритм.
